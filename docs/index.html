LifeOS

Multi-agent personal optimization, with a Judge that actually says no.

Three specialised AI agents collaborate to find real job matches, plan your day, and brief your focus. A separate Judge agent verifies every output against an explicit contract — and rejects fabrications, contradictions, and overconfident claims before they reach the user.

What it does

1. Real job search

Career Agent queries Arbeitnow + Remotive + RemoteOK + ejobs.ro (Romania) — plus optional Adzuna and LinkedIn — then picks one seniority-appropriate match. Returns an English overview and a link. You apply yourself.

2. Day plan

Schedule Agent takes your fixed events, mood, sleep, and energy — and produces a realistic time-blocked plan with the recommended learning task placed sensibly.

3. Productivity briefing

Productivity Agent reads the accepted plan and outputs one focus rule, your top-priority block, a procrastination-risk level, and a 1-10 productivity score.

4. Judge gates everything

A separate Judge agent on a different model tier runs a deterministic contract check, then an LLM consistency check. Outputs that fabricate or contradict get rejected before reaching the user.

The Judge runs on Gemini 3.5 Flash while the agents run on Gemini 3.1 Flash-Lite. Same-model self-grading is a known weakness in LLM-as-judge research, so the architecture explicitly avoids it.

Design choices, defended

3 agents, not six

The original brief had six agents. We deliberately cut to three so each does its job under a real contract — depth over breadth.

Index-based job selection

Career Agent returns an integer 1..N (or 0). The orchestrator fills the verbatim job details. Hallucinating a job is structurally impossible.

Contracts as YAML

Behaviour rules live in config files the Judge loads at runtime — not hardcoded prompts. Easy to inspect, easy to extend.

Two-layer Judge

Deterministic check first (free, instant, catches structural errors). LLM consistency check only when the deterministic pass succeeds.

Human-in-the-loop

The system proposes; the user decides. No autonomous actions, no applications, no calendar writes, no irrecoverable side-effects.

Cheap by construction

Free Gemini API tier + free Streamlit deploy + free GitHub Pages + free job APIs. Total runtime cost: zero.

n8n integration — daily Telegram delivery

LifeOS ships with a FastAPI wrapper (api.py) so it can be called as an HTTP endpoint. An n8n workflow (Cron → Telegram ask → Google Calendar fetch → HTTP POST to LifeOS → Telegram reply) wraps the system into a daily morning automation. Runs on a self-hosted n8n in Docker — no third-party cloud orchestration needed.
